# Go 编程语言知识地图

```
Go 语言
```

---

## 1️⃣ 基础语法 (2周)

### 1.1 类型与变量
- ⏱️ **1天** | 🎯 **定义数据结构的基础**
  - 基本类型：int, float, string, bool
  - 复合类型：array, slice, map, struct
  - 指针理解（非必须但建议）
  - 🔗 **无前置依赖**

### 1.2 控制流
- ⏱️ **2天** | 🎯 **编写逻辑分支与循环**
  - if/else，switch（无括号写法）
  - for（Go 唯一的循环关键字）
  - defer（资源清理惯用模式）
  - 🔗 **依赖：1.1**

### 1.3 函数与方法
- ⏱️ **3天** | 🎯 **代码复用与组织**
  - 函数定义：多返回值、命名返回值
  - 方法：receiver 理解（值 vs 指针）
  - 闭包与匿名函数
  - 🔗 **依赖：1.1, 1.2**

### 1.4 接口
- ⏱️ **3天** | 🎯 **解耦与多态**
  - 接口定义与隐式实现
  - 空接口 interface{} 与类型断言
  - 常用标准库接口（io.Reader/Writer, error）
  - 🔗 **依赖：1.3**

---

## 2️⃣ 并发模型 (2周)

### 2.1 Goroutine
- ⏱️ **2天** | 🎯 **轻量级并发任务**
  - go 关键字启动
  - Goroutine 调度原理（M:N 模型概念）
  - 常见启动模式
  - 🔗 **依赖：1.3**

### 2.2 Channel
- ⏱️ **4天** | 🎯 **Goroutine 间通信**
  - 创建与基本操作（send/receive/close）
  - 缓冲 vs 无缓冲
  - select 多路复用
  - range 遍历 channel
  - 🔗 **依赖：2.1**

### 2.3 并发模式
- ⏱️ **4天** | 🎯 **解决实际并发问题**
  - Worker Pool（任务分发）
  - Fan-In/Fan-Out
  - Context 超时与取消
  - 常见竞态条件与 sync 包（Mutex, WaitGroup）
  - 🔗 **依赖：2.2**

---

## 3️⃣ 工程实践 (2周)

### 3.1 项目结构
- ⏱️ **2天** | 🎯 **组织可维护代码**
  - 标准 go mod 布局
  - 内部包与可见性（大小写规则）
  - 常见目录约定（cmd/, pkg/, internal/）
  - 🔗 **依赖：1.4**

### 3.2 错误处理
- ⏱️ **2天** | 🎯 **优雅处理异常**
  - error 接口与惯用写法
  - errors.Wrap 与错误链
  - panic/recover（少用但需理解）
  - 🔗 **依赖：1.4**

### 3.3 单元测试
- ⏱️ **3天** | 🎯 **保证代码质量**
  - testing 包与 _test.go 文件
  - 表驱动测试
  - mock 与 testify
  - 基准测试（benchmark）
  - 🔗 **依赖：3.1**

### 3.4 常用标准库
- ⏱️ **3天** | 🎯 **避免重复造轮子**
  - net/http：HTTP 服务端与客户端
  - encoding/json：JSON 序列化
  - os/io/filepath：文件操作
  - log/slog：结构化日志
  - 🔗 **依赖：1.4**

## 4️⃣ 生态工具 (1周)

### 4.1 Go Modules
- ⏱️ **1天** | 🎯 **依赖管理**
  - go.mod 与 go.sum
  - 常用命令：go mod tidy/get/updage
  - 私有仓库代理
  - 🔗 **依赖：3.1**

### 4.2 命令行工具
- ⏱️ **1天** | 🎯 **日常开发效率**
  - go run/build/test/install
  - go fmt（自动格式化）
  - go vet（静态检查）
  - gofmt/s（格式化选项）
  - 🔗 **无前置依赖**

### 4.3 常用框架
- ⏱️ **3天** | 🎯 **快速构建应用**
  - Web 框架：gin/echo/fiber（选一）
  - ORM：gorm/sqlc（选一）
  - 配置管理：viper
  - CLI 工具：cobra
  - 🔗 **依赖：3.4, 4.1**
