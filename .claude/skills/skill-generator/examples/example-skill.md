以下是一个由 skill-generator 生成的完整技能示例，供参考：

---

假设用户输入："帮我创建一个技能，用于快速总结任何文章的核心要点，输出为 markdown 文件"

生成的技能如下：

---

## 文件：`.claude/skills/article-summarizer/SKILL.md`

```markdown
---
name: article-summarizer
description: 快速总结任何文章的核心要点，生成精炼的摘要文档。当用户提供文章内容或文章链接，想要快速获取核心要点时使用。
---

快速提取任意文章的核心要点，生成结构化的精炼摘要，帮助用户在最短时间内掌握文章精华。

## 核心原则

1. **精准提炼**：只提取文章的核心论点和关键信息，去除冗余描述和过渡性内容
2. **结构化输出**：摘要必须按逻辑结构组织，而非简单罗列
3. **忠实原文**：不添加原文中没有的观点，不曲解作者意图
4. **适度精炼**：摘要长度控制在原文的 10%-20%，保留所有关键信息点

## 工作流程

### 第一步：获取文章内容

1. 如果用户直接提供了文章文本，直接进入第二步
2. 如果用户提供了文章链接，使用 `agent-browser` 技能访问链接获取内容
3. 确认文章内容已完整获取

### 第二步：分析文章结构

1. 识别文章的主题和核心论点
2. 梳理文章的论证逻辑和段落结构
3. 标记关键数据、案例和结论

### 第三步：生成摘要

1. 按照【输出结构】组织摘要内容
2. 确保每个要点都有原文依据
3. 检查摘要是否覆盖了文章所有核心信息

### 第四步：输出

1. 将摘要保存为 markdown 文件
2. 保存位置和命名参照【输出】模块

## 输出

- **格式**：Markdown
- **命名**：`{文章主题关键词}摘要.md`
- **位置**：当前项目 `tmp` 目录下
- **内容结构**：
  - 一句话概括：用一句话总结文章核心观点
  - 核心要点：按逻辑顺序列出 3-7 个核心要点
  - 关键数据/案例：列出文章中的重要数据或案例（如有）
  - 作者结论：原文作者的最终结论

## 绝对不能做的事

1. 不要编造原文中不存在的信息
2. 不要遗漏文章的核心论点
3. 不要添加个人评价或主观解读
```

---

## 说明

以上示例展示了 skill-generator 生成技能时的标准格式：
- **Frontmatter**：包含 `name` 和 `description`
- **概述**：一句话说明技能目标
- **核心原则**：编号列表 + 加粗关键字 + 冒号 + 详细描述
- **工作流程**：分步骤，每步有子步骤
- **输出**：明确格式、命名、位置和内容结构
- **绝对不能做的事**：约束 AI 行为边界
