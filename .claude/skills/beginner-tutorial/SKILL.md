核心目标是为零基础/小白用户，生成快速上手的Demo/教程。以达到快速上手的目的。

## 核心原则

1️⃣ **只求"上手"，不求"理解原理"**
   - 不讲底层原理、历史背景、进阶知识
   - 只告诉他"现在你只要这么做，就能跑起来"

2️⃣ **必须极其简单**
   - 如果有2种做法，只选最简单的那一种
   - 如果一个步骤可以省略，就直接省略
   - 如果一个名词不是"必须立刻用到"，就不要出现

3️⃣ **傻瓜式操作**
   - 每一步都要拆到"点哪里 / 输入什么 / 看到什么结果"
   - 不允许出现"自行配置 / 参考文档 / 按需调整"这种说法

4️⃣ **所有第一次出现的名词，都必须立刻解释**
   - 用一句大白话解释
   - 类比现实生活
   - 禁止使用专业术语解释专业术语

5️⃣ **语言要求**
   - 像在教一个完全不懂技术的朋友
   - 用短句、口语化
   - 不要假设用户"应该知道"

## 操作步骤

### 第1步：确认学习主题

询问用户想学习什么技术/知识。

### 第2步：生成教程内容

严格按照以下结构生成教程：

```markdown
==============================
一、我们最终要做成什么？
- 用一句话说明最终效果
- 告诉用户："等下你会得到什么"

二、开始前你需要准备什么？
- 只列【100% 必须】的东西
- 每一项都说明：是什么 + 怎么获得
- 如果什么都不需要，就明确说「你什么都不用准备」

三、一步一步照着做（核心部分）
- 用「第 1 步 / 第 2 步 / 第 3 步」
- 每一步包含：
  - 你要做什么（非常具体）
  - 你为什么现在要做这个（一句话）
  - 做完后你应该看到什么结果

四、恭喜，你已经上手了
- 明确告诉用户：你已经完成了入门
- 说明他"已经掌握了什么能力"（哪怕很初级）

五、如果你想继续（可选）
- 只给 1～2 个"下一步方向"
- 不展开讲
==============================
```

### 第3步：保存到文件
将生成的教程保存到当前项目目录的 `learn` 文件夹中，文件名格式为：`{主题}上手.md`


## 重要提醒

- **简单 > 完整**
- **能跑 > 能懂**
- **上手 > 一切**
