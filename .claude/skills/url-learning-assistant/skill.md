# 任务
根据用户的url学习策略，从用户提供的url中爬取、收集相关信息，然后按照url的学习策略方式对网页内容进行消化、整理后完成输出，从而帮助用户更好的学习任何网页资源内容；

## url学习策略
主要分为三种：
1. 快速了解：快速的了解网页的主要内容，主要回答它是什么、能解决什么问题、有什么特点等等
2. 仅学习当前页面：仅学习当前页面的内容，不进行其他操作
3. 按照特定主题/模块/章节等进行学习：按照用户指定的主题、模块、章节等进行学习

## 工作流程
1. 确定url学习策略
   - 如果从用户的描述中能确定url学习策略，那么直接进入下一步
   - 如果不能确定学习策略，那么需要向用户询问学习策略，询问采用`AskUserQuestion`工具提问，提问内容为：
     - "请告诉我您希望如何学习这个url？"
       - "a. 只看当前页面: 仅学习当前页面的内容，不进行其他操作"
       - "b. 快速了解: 快速的了解网页的主要内容，主要回答它是什么、能解决什么问题、有什么特点等等"
       - "c. 按照模块学习: 按照用户指定的主题、模块、章节等进行学习"
       - "d. 其他: 用户自定义需求"

2. 根据学习策略处理方式处理，详见【学习策略处理方式】

## 学习策略处理方式
### 1. 快速了解
1. 核心任务：
  对网页进行快速了解，主要回答它是什么、能解决什么问题、有什么特点等等
2. 网页爬取: 
    - 使用 `agent-browser` skill 访问 URL，获取页面内容
    - 如果有必要，可以简单的逛逛其它相关页面，以完成信息的补充完善；
    - 选择是否逛其他页面的依据是，当前页面的内容是否能满足核心任务的要求
3. 输出:
    - 根据网页爬取到的内容，按照核心任务的要求进行整理和输出

### 2. 仅学习当前页面
1. 核心任务： 
  抓取当前页面的内容，然后整理消化后，以用户易于理解的方式输出
2. 网页爬取: 
    - 使用 `agent-browser` skill 访问 Url获取页面内容
3. 输出:
    - 对爬取到的网页内容，进行汇总、消化、整理，以方便用户阅读和理解输出
    - 输出的时候，不能随意丢弃页面上的关键信息内容，尽量保证内容的不丢弃、完整
    - 易于理解的方式输出，指的是输出的内容要条理清晰、重点突出

### 3. 按照特定主题/模块/章节等进行学习
核心任务：
  从用户提供的url中爬取，用户想要学习的主题/模块/章节内容，然后汇总、整理、消化后，以易于理解的方式输出

流程：
1. 确定用户想要学习的主题/模块/章节等
  - 使用 `agent-browser` skill 访问用户指定的url
  - 分析整理后，提供一个大概的主题/模块/章节列表，然后供用户选择
  - 用户选择后，记录用户选择的主题/模块/章节
2. 网页爬取：
  - 使用 `agent-browser` skill 访问用户指定的url
  - 快速找到和用户选择的主题/模块/章节/内容相关的链接，可以从页面的导航栏、侧边栏、目录、网页跳转链接等位置查找
  - 访问这些链接获取详细信息，进行信息收集
3. 输出：
  结合2中收集到的网页内容，为用户编写一个实战、可以实操的教学内容
  - 教学内容，主要体现在实际环境中要怎么使用、它的使用场景
  - 教学内容应该包含：背景介绍、使用方法、实际案例、注意事项等
  - 生成的教学内容需要使用 markdown 格式，输出到当前项目tmp目录下，以{域名}_{主题/模块/url}.md命名

## 绝对不能做的事
1. 直接使用`Fetch`等其它工具直接访问网页，必须使用`agent-browser` skill 做网页访问
2. 不要再单独委派agent 去完成任务

## 输出原则
- 耐心、细致，掰开揉碎
- 用通俗语言解释专业概念
- 结构化呈现，便于理解和记忆
