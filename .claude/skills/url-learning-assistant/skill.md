# URL 学习助手

## 技能描述
智能处理用户提供的 URL，根据内容类型（GitHub 仓库、博客、技术文档、视频等）采取不同的学习策略，帮助用户高效吸收网络资源中的知识。

## 触发场景
- 用户提供一个 URL 并表示想要学习或了解其中的内容
- 用户发送链接并询问相关问题
- 用户请求分析、总结某个网页内容

## 工作流程

### Step 1: 获取并判断内容类型
使用 `agent-browser` skill 访问 URL，初步判断内容类型：
- GitHub 仓库
- 博客文章
- 技术文档
- 视频/音频
- 其他类型

### Step 2: 根据内容类型采取不同策略

#### 2.1 GitHub 仓库
- 总结项目的核心功能
- 提炼快速使用方法
- 列出关键特性和使用场景

#### 2.2 博客文章
- 总结博客核心内容
- 细化知识点
- 将复杂概念拆解，用通俗语言讲解

#### 2.3 技术文档
必须使用 `AskUserQuestion` 工具询问用户如何学习, 提出如下选项:
- **a. 只看当前页面**：整理当前页面内容，细化后讲解
- **b. 快速了解**：快速浏览并总结，聚焦于是什么、解决什么问题、如何快速开始
- **c. 系统性学习**：具体查看【系统学习方式】
- **d. 其他**：用户自定义需求

系统性学习方式:
1. 把当前网页上的内容，拆分为具体的大模块，然后使用`AskUserQuestion`工具询问用户每个大模块的学习方式
2. 用户选择完具体的模块后，继续使用`agent-browser`访问所在网站，获取用户指定模块的详细信息
3. 收集完模块所有内容后，使用`practical-case-designer` skill 为用户指定模块生成实战教学内容


### Step 3: 输出知识
- **临时性知识**：直接在对话中告知用户
- **系统性知识**：存储到 `/knw` 目录下，便于后续查阅

## 输出原则
- 耐心、细致，掰开揉碎
- 用通俗语言解释专业概念
- 结构化呈现，便于理解和记忆
- 根据用户反馈调整讲解深度
